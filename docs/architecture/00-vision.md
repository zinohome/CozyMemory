## 🎯 项目定位

### 从"平台"到"库"的转变

**v0.1 定位 (过度设计)**:
- ❌ 独立 API 服务
- ❌ 多引擎路由平台
- ❌ 缓存层 + 智能路由
- ❌ 需要部署和运维

**v0.2 定位 (简化务实)**:
- ✅ Python 库 (`pip install cozy-memory`)
- ✅ 轻量级记忆管理工具
- ✅ 直接集成到现有项目
- ✅ 零部署成本

## 📐 设计原则

### 1. 库优先 (Library First)

```python
# 简单直接
from cozy_memory import MemoryService

service = MemoryService()
service.add("用户喜欢咖啡", source="manual")
```

### 2. 集成优先 (Integration First)

- 无需独立服务
- 无需数据库迁移
- 无需配置管理
- 开箱即用

### 3. 渐进式复杂度 (Progressive Complexity)

```
Level 1: 简单存储 (SQLite)
    ↓
Level 2: 多适配器 (可选)
    ↓
Level 3: 智能路由 (可选)
```

### 4. 开发者体验 (DX First)

- 类型注解完整
- 文档清晰简洁
- 错误信息友好
- 示例丰富

## 🦀 维护者注释

**为什么从平台转向库？**

1. **实际需求**: 用户需要的是"能用的记忆工具"，不是"另一个微服务"
2. **降低门槛**: `pip install` 比 Docker 部署简单 10 倍
3. **快速迭代**: 库的发布周期比服务快
4. **生态兼容**: 更容易集成到现有项目

**架构决策背后的思考**:

- **移除 API Gateway**: 库不需要网关，直接函数调用
- **移除 GraphQL**: RESTful 足够，避免过度设计
- **移除 gRPC**: 本地调用不需要 RPC
- **简化数据模型**: 聚焦核心功能，避免功能蔓延

**未来演进原则**:

> "每个新功能的添加，都要问：这个功能是否让简单用例变复杂？"
