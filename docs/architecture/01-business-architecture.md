## 1. 业务概述

### 1.1 业务愿景

构建**统一 AI 记忆服务平台**，整合业界领先的记忆引擎 (Mem0、Memobase、Cognee)，为企业和开发者提供一站式记忆服务，降低 AI 应用开发门槛，提升用户体验。

### 1.2 业务目标

| 目标 | 描述 | 成功指标 |
|------|------|---------|
| **统一入口** | 提供统一的 API 入口，屏蔽底层引擎差异 | API 调用量 > 100 万/月 |
| **智能路由** | 根据场景自动选择最优引擎 | 路由准确率 > 85% |
| **降低成本** | 通过批量处理和缓存降低 LLM 成本 | 成本降低 50% |
| **提升性能** | 通过缓存和并行调用降低延迟 | P95 < 200ms |
| **保证质量** | 通过融合和去重提升结果质量 | 用户满意度 > 4.5/5 |

### 1.3 业务范围

**纳入范围**:
- ✅ RESTful API (基础 CRUD)
- ✅ GraphQL API (灵活查询)
- ✅ gRPC API (高性能/流式)
- ✅ 用户认证和授权
- ✅ 智能路由 (规则 + LLM)
- ✅ 结果融合 (去重、排序)
- ✅ 缓存管理
- ✅ 监控和日志

**不纳入范围 (Phase 1)**:
- ❌ 前端管理界面
- ❌ 可视化图谱工具
- ❌ 离线批处理
- ❌ 多语言 SDK (仅 Python)

## 2. 业务流程

### 2.1 核心业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                    记忆管理业务流程                          │
└─────────────────────────────────────────────────────────────┘

1. 用户创建记忆
   │
   ├─→ 2. API Gateway 接收请求
   │   ├─→ 认证验证 (JWT/API Key)
   │   └─→ 请求体验证
   │
   ├─→ 3. Router Service 路由决策
   │   ├─→ 规则匹配 (关键词/模式)
   │   └─→ LLM 意图识别 (可选)
   │
   ├─→ 4. Adapter Layer 并行调用引擎
   │   ├─→ Mem0 (事实提取)
   │   ├─→ Memobase (画像更新)
   │   └─→ Cognee (知识构建)
   │
   ├─→ 5. Fusion Service 结果融合
   │   ├─→ 去重
   │   ├─→ 评分
   │   └─→ 排序
   │
   └─→ 6. 返回统一格式结果

```

### 2.2 记忆查询流程

```
1. 用户查询记忆
   │
   ├─→ 2. 检查缓存 (Redis)
   │   ├─→ Hit: 直接返回 (<5ms)
   │   └─→ Miss: 继续向下
   │
   ├─→ 3. Router Service 路由决策
   │   ├─→ 分析查询意图
   │   └─→ 选择引擎
   │
   ├─→ 4. 并行查询引擎
   │   ├─→ Mem0: 向量搜索
   │   ├─→ Memobase: SQL 查询
   │   └─→ Cognee: 图谱遍历
   │
   ├─→ 5. Fusion Service 融合结果
   │   ├─→ 合并
   │   ├─→ 去重
   │   ├─→ RRF 排序
   │   └─→ 截断
   │
   ├─→ 6. 写入缓存 (异步)
   │
   └─→ 7. 返回结果
```

### 2.3 用户管理流程

```
1. 用户注册
   │
   ├─→ 2. 验证邮箱和密码
   │
   ├─→ 3. 创建用户记录
   │   └─→ 哈希密码 (bcrypt)
   │
   ├─→ 4. 返回 JWT Token
   │
   └─→ 5. 用户可创建 API Key

```

## 4. 用例分析

### 4.1 核心用例

#### UC-001: 创建记忆

**参与者**: 开发者  
**前置条件**: 用户已认证  
**后置条件**: 记忆已存储

**主流程**:
1. 开发者调用 `POST /api/v1/memories`
2. 系统验证认证信息
3. 系统验证请求体
4. 系统路由到合适引擎
5. 引擎存储记忆
6. 系统返回记忆 ID

**备选流程**:
- 4a. 路由失败 → 使用默认引擎
- 5a. 引擎不可用 → 降级到备用引擎

**异常流程**:
- 2a. 认证失败 → 返回 401
- 3a. 验证失败 → 返回 400

---

#### UC-002: 查询记忆

**参与者**: 开发者  
**前置条件**: 用户已认证  
**后置条件**: 返回记忆列表

**主流程**:
1. 开发者调用 `GET /api/v1/memories/search?query=xxx`
2. 系统验证认证信息
3. 系统检查缓存
4. 缓存未命中，路由到合适引擎
5. 并行查询多个引擎
6. 融合结果
7. 返回记忆列表

**备选流程**:
- 3a. 缓存命中 → 直接返回
- 4a. 无合适引擎 → 查询全部引擎

---

#### UC-003: 删除记忆

**参与者**: 开发者  
**前置条件**: 用户已认证，记忆存在  
**后置条件**: 记忆已删除

**主流程**:
1. 开发者调用 `DELETE /api/v1/memories/{id}`
2. 系统验证认证信息
3. 系统验证记忆所有权
4. 系统软删除记忆
5. 系统记录删除日志
6. 返回成功

**备选流程**:
- 3a. 无所有权 → 返回 403
- 3b. 记忆不存在 → 返回 404

---

#### UC-004: 创建 API Key

**参与者**: 开发者  
**前置条件**: 用户已登录  
**后置条件**: API Key 已创建

**主流程**:
1. 开发者调用 `POST /users/me/api-keys`
2. 系统验证 JWT Token
3. 系统生成 API Key
4. 系统存储 API Key 哈希
5. 返回 API Key (仅显示一次)

**备选流程**:
- 2a. Token 过期 → 返回 401
- 4a. 达到上限 → 返回 429

## 5. 业务场景

### 5.1 场景 1: AI 助手对话

**背景**: 构建个性化的 AI 助手，需要记住用户偏好和历史对话。

**需求**:
- 实时记忆用户偏好 (Mem0)
- 长期记忆用户画像 (Memobase)
- 快速响应 (< 200ms)

**解决方案**:
```python
# 对话流程
user: "我喜欢 Python 编程"
→ Mem0 存储事实
→ Memobase 更新画像

user: "我记得我喜欢什么？"
→ 路由到 Memobase
→ 返回画像信息
→ AI 助手回答："您喜欢 Python 编程"
```

---

### 5.2 场景 2: 企业知识库

**背景**: 企业需要构建知识库，支持员工查询公司文档和政策。

**需求**:
- 文档解析和知识抽取 (Cognee)
- 图谱关系推理
- 权限控制

**解决方案**:
```python
# 知识管理流程
上传员工手册.pdf
→ Cognee 解析文档
→ 构建知识图谱
→ 存储实体和关系

员工查询："年假有多少天？"
→ Cognee 图谱搜索
→ 返回相关政策
```

---

### 5.3 场景 3: 个性化推荐

**背景**: 电商平台需要基于用户历史和偏好进行个性化推荐。

**需求**:
- 用户行为记录 (Memobase)
- 偏好分析 (Mem0)
- 低成本 (批量处理)

**解决方案**:
```python
# 推荐流程
用户浏览商品 A, B, C
→ Memobase 记录行为
→ 批量处理 (夜间)
→ 更新用户画像

用户访问首页
→ 查询用户画像
→ 推荐相似商品
```
