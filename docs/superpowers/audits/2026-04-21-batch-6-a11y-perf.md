# Batch 6 — A11y + Perf Audit

Date: 2026-04-21
Scope: CozyMemory UI (Next.js 16 / React 19) — 9 user-facing pages

## A11y (axe-core 4.10.2)

### Scan methodology
Real-navigation to each route, inject axe-core via CDN, run with `resultTypes:
['violations']`. Pages covered: `/dashboard`, `/memory`, `/profiles`,
`/knowledge`, `/context`, `/playground`, `/users`, `/backup`, `/settings`.

### Baseline findings
| Page | Violation | Impact | Count |
|---|---|---|---|
| all 9 | `region` — sidebar content not in landmark | moderate | 13 per page |
| dashboard | `heading-order` — h1 → h3 skip | moderate | 1 |
| memory | `button-name` — icon-only search btn + Select trigger | **critical** | 2 |
| context | `button-name` — 2 Select triggers | **critical** | 2 |
| playground | `button-name` — Send btn + Select | **critical** | 2 |
| users | `color-contrast` — th + amber warning text | serious | 3 |
| users | `empty-table-header` | minor | 1 |
| backup | `button-name` — user-selector Select | **critical** | 1 |

### Fixes
1. **region**: `AppSidebar` → add `role="navigation" aria-label="Main"` (props
   forward to shadcn Sidebar's `data-slot="sidebar-container"`)
2. **heading-order** dashboard: 4× `<h3>` section titles → `<h2>` (H1 Dashboard
   → H2 sections)
3. **button-name** all Select triggers: add `aria-label` on `SelectTrigger`
   in `user-selector.tsx`, `context/page.tsx`, `knowledge/page.tsx`,
   `playground/page.tsx` (×2)
4. **button-name** icon-only `<Button>`s: Memory search btn, Playground
   Send/Stop btns
5. **color-contrast** users: `text-muted-foreground` → `text-foreground` on
   table `<th>`s; `text-amber-600/400` → `text-amber-700/300`
6. **empty-table-header**: add `<span className="sr-only">Actions</span>` to
   delete-column th

### Post-fix result
All 9 pages: **0 violations** (axe-core scan post-rebuild, 2026-04-21).

## Perf

### Build baseline (Next 16 / Turbopack)
- `.next/static/chunks/` total: **1.8 MB** (unchanged post-fixes; memo is
  runtime-only)
- Top 5 chunks: 228 K, 184 K, 142 K, 110 K, 87 K (minified, hashed names)
- 14 routes, 12 pre-rendered static, 1 dynamic (`/api/chat`)

### Observations
- **react-force-graph-2d** already dynamic-imported via `next/dynamic` in
  `knowledge-graph.tsx` with `ssr: false` — no action
- **lucide-react** (40 distinct icons) imported named, tree-shaken — no action
- **date-fns** not used directly (transitive dep); tree-shaking excludes it
- **Turbopack** hashed chunks; dep-to-chunk mapping is approximate without
  `@next/bundle-analyzer` (not installed for this pass — bundle dominated by
  React + Next framework code, not app code)

### Fixes
Long-list row components were re-rendering on unrelated parent state changes
(input typing, toast state, mutation flags). Wrapped in `React.memo` with
stable `useCallback` delete handlers:

- `MemoryRow` in `memory/page.tsx` — list of up to N conversation memories
- `ProfileItemRow` in `profiles/page.tsx` — list of N profile topics

Both pages pass a stable `handleDelete` via `useCallback` to prevent memo
invalidation.

### Not done (documented, not fixed)
- `@next/bundle-analyzer` visualization — skipped; bundle already <2 MB and
  dominated by framework. Revisit if total exceeds 4 MB.
- Sparkline `React.memo` — `values[]` identity changes every 10 s poll, memo
  won't help without expensive deep compare. Internal `useMemo` for path calc
  is sufficient.
- Pre-existing React 19 lint warnings (`set-state-in-effect` in `use-mobile.ts`
  and `use-hotkeys.ts` ref-in-render) — out of batch 6 scope, affect only
  future hot-path investigations.
